Data Layer (Demo)

Dataset Selection

Analyzed 7 CSV files. Selected Amazon Sale Report (128K rows) for clean schema, complete transactions, and consistent date range (Apr-Jun 2022).

Why Single Table?

Other files had: mixed schemas, embedded tables, date range mismatch, zero SKU overlap, or non-tabular format. No meaningful joins possible.

Processing Approach

Raw CSV loaded directly into DuckDB with column renaming. No ETL pipeline needed for demo - data is clean enough for analytics.

Table: amazon_sales

order_id VARCHAR
order_date DATE
status VARCHAR
category VARCHAR
size VARCHAR
quantity INTEGER
amount DECIMAL
city VARCHAR
state VARCHAR
is_b2b BOOLEAN
